整理了一下最喜欢的前端 UI 框架 Bootstrap 的发展史
从开始接触到现在一直都是 3.x
还不知道短短不到 5 年的时间竟发生了翻天覆地的变换
期待着 4.x 的完善
Bootstrap 可以说是前端开发人员比较喜欢的 UI 框架，同样也受后台的欢迎

Bootstrap

  • Bootstrap 的更新日志
  • Bootstrap 是 Twitter 在2011年8月推出的开源 WEB 前端框架
  • 在Bootstrap 2 中，对框架中的某些关键部分增加了对移动设备友好的样式
  • Bootstrap 3 中，重写了整个框架，使其一开始就是对移动设备友好的
  • 这次不是简单的增加一些可选的针对移动设备的样式，而是直接融合进了框架的内核中
  • Bootstrap3 是移动设备优先的
  • 目前 Bootstrap 4 还处于 alpha 测试版，官方文档都是预览版

Bootstrap 2

  • 发布2.0.1版
  • 2012年2月份就再次放出新版本2.0
  • jquery/
  • 基于 Less 框架构建的 CSS
  • 2.0.1 支持几乎所有的主流浏览器，包括Safari、Chrome，Firefox、IE7.0及以上，最新版本还支持智能手机 WebUI
  • 2.0版本采用了更灵活也更受欢迎的12栏网格布局，并以此来实现其各种布局框架
  • 增加了响应式设计，以适应各种移动终端的需求
  • 完善和改进原有样式库，并提供更丰富的新样式，包括样式繁多的图标（icon），漂亮易用的进度条等
  • 改进和增加了自定义 jQuery 插件，完善文档，修复 bug
  • 同时还提供了很多基于 Bootstrap 构建的网站样例

Bootstrap 3

  • Bootstrap 3 发布
  • 2013年8月19日发布
  • jquery/1.11.3
  • Bootstrap 3并不向后兼容Bootstrap v2.x
  • 再见，IE7！及Firefox3.6 。最低从IE8开始支持。而且因为设计已经变成移动优先，所以对IE8的支持也需要respond.js
  • 所有元素默认盒模型（box model）使用了 border-box，注意，实现的方法是非常暴力的设置 * { box-sizing: border-box; }，理由呢，就是方便计算，这也是它可能会和其他第三方框架发生冲突的主要原因

Bootstrap 4

  • Bootstrap-4-alpha
  • 在2015年8月19日发布
  • jquery/2.1.4
  • 从 Less 迁移到 Sass
  • 改进栅栏系统
  • 缺省弹性框支持
  • 缩略图、卡片面板
  • 合并所有 HTML resets 到一个新的模块中：Reboot
  • 全新自定义选项
  • 彻底不再支持 IE8
  • 重写所有的 JavaScript 插件
  • 改进工具提示和 popovers 的自动定位
  • 改进文档
  • 其他大量改进
