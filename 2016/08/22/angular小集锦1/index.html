监听路由跳转

在一个控制器里做业务逻辑处理时，遇到一个需求
需要定时向服务器请求数据，于是自然就用到了定时器
可是问题又来了，当进行路由跳转时，虽然已经清除了之前的控制器
然而并没有清除该控制器中的定时器
angular 自然会想到这个问题
于是用到了监听路由跳转机制

强制使 angular 将值进行绑定

有时候在控制中处理业务逻辑时
动态改变 $scope 上绑定的值，然而页面上过了一会才进行渲染
于是就需要进行改变值强制改变

解除 angular 不信任

有时候需要进行动态获取并添加 html 字符串
首先想到的就是 ng-bind-html
这时候 angular 就有一个防 XSS 的解析机制
对于 angular 不信任的 html 字符串无法进行绑定
就需要手动解除不信任
具体什么原理，不晓得
